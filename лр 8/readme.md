1     **ВАРИАНТ 16. Жадный алгоритм для упаковки контейнеров (Bin Packing)**

Задача: реализовать несколько жадных стратегий для упаковки предметов в контейнеры.
Требования:
- Стратегия "First Fit" (FF)
- Стратегия "First Fit Decreasing" (FFD)
- Стратегия "Best Fit"
- Сравнить результаты всех трех стратегий


## 2 Описание алгоритмов

- First Fit (FF):
  - Для каждого предмета проходит по списку контейнеров слева направо.
  - Ставит предмет в первый контейнер, куда он помещается.
  - Если такой контейнер не найден, создаёт новый.

- First Fit Decreasing (FFD):
  - Сначала сортирует предметы по убыванию размера: \(O(n \log n)\).
  - Потом применяет алгоритм First Fit к отсортированному списку.

- Best Fit (BF):
  - Для каждого предмета выбирает контейнер, в котором после помещения предмета останется минимальное свободное место.
  - Если подходящего контейнера нет, создаёт новый.

  ## 3  Временная сложность
  
    O(n^2)
  
  ## 4 Оценка временной сложности
First Fit (FF):
Каждая итерация проходит по массиву контейнеров (до n контейнеров), что даёт нам вложенный цикл.
Следовательно, временная сложность алгоритма First Fit — O(n2)O(n 2 ).

First Fit Decreasing (FFD):
Сначала происходит сортировка массива предметов, что добавляет временную сложность O(nlogn)O(nlogn).
Далее алгоритм действует аналогично First Fit, что даёт общую временную сложность: O(nlogn+n2)O(nlogn+n 2), что упрощенно становится O(n2)O(n 2) при больших n.

Best Fit (BF):
Здесь также используется цикл по предметам и поиск подходящего контейнера, аналогичный First Fit.
Поэтому общая временная сложность также составляет O(n2)O(n 2).
 

  ## 5 Ответ на контрольный вопрос

**_16_**. Оператор соседства
Что такое оператор соседства в контексте эвристических алгоритмов? Приведите примеры.

_Оператор соседства_ — это процедура, позволяющая изменять текущее состояние (решение) системы таким образом, чтобы перейти в одно из соседних состояний (решений). Другими словами, это операция, преобразующая текущее решение в другое близкое к нему решение, называемое соседним решением.

Использование операторов соседства характерно для многих эвристических алгоритмов, таких как генетический алгоритм, метод локального поиска и симулированное отжигание. Эти операторы позволяют исследовать окрестности текущего решения, перемещаясь в пространстве решений и стремясь приблизиться к глобальному оптимуму.

Примеры:

Типичные примеры операторов соседства:
Перестановка (Swap):
Изменяется местами два случайно выбранных элемента в решении.
Пример: пусть дано расписание экзаменационных сессий студентов, оператор перестановки меняет дни двух экзаменов местами.

Изменение направления пути (Inversion):
Меняется направление отрезка маршрута между двумя случайными точками.
Пример: в задаче коммивояжера меняется порядок прохождения части маршрута.

Замещение (Replace):
Один элемент заменяется другим элементом из заранее заданного множества.
Пример: замена одного типа оборудования на другое при составлении производственного расписания.
